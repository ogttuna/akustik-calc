- subtract wall user-material formula companion completeness, field
  basis integrity, field target-output independence, timber-stud/CLT
  field target-output independence, and timber-stud/CLT building
  lab-companion basis integrity because those runtime owners and
  coverage refreshes are now closed;
- subtract floor visible-floating and low-density exact ASTM companion
  chains that already landed owner plus refresh unless a new request
  shape moves more target outputs than the wall candidates;
- subtract direct-fixed and compatible-anchor wall A-weighted/lab
  companion chains already protected by their coverage refreshes.

Iteration 2 - runtime-first candidate families:

- compare wall double-leaf/framed user-material adjacent widening only
  if it can open a physically owned request shape rather than silently
  defaulting missing topology or absorber inputs;
- compare wall opening/leak or common-wall residual work only if the
  same-basis owner proof is complete enough to move runtime values
  without broad source crawling;
- compare floor open-box/open-web timber residuals only if they move
  actual `Ln,w`, `DeltaLw`, `L'n,w`, `L'nT,w`, `IIC`, or `AIIC`
  support through an owned formula or exact/compatible anchor;
- compare timber-stud and CLT formula retuning only if there is bounded
  measured evidence or a locally owned formula basis, not just a desire
  to improve a scalar.

Iteration 3 - boundary safety:

- reject scalar aliasing across `Rw`, `STC`, `C`, `Ctr`, `IIC`, and
  `AIIC`;
- reject building or field outputs without complete context and an
  owned adapter;
- reject work that makes an unsupported or `needs_input` route look
  calculable by defaulting missing physical fields;
- reject UI/report-only work even if it would make the result easier to
  display.

Iteration 4 - expected owner selection:

- prefer a candidate with `newCalculableRequestShapes >= 2`,
  `newCalculableTargetOutputs >= 5`, `runtimeBasisPromotions >= 2`, or
  a reproduced wrong-answer / wrong-basis fix for a common construction;
- apply the readiness gates in
  `docs/calculator/HIGH_ROI_CALCULATOR_IMPLEMENTATION_SELECTION_PLAN_2026-06-18.md`
  before naming the owner: route basis, physical inputs, target
  movement, metric/basis integrity, bounded blast radius, and evidence
  confidence;
- if no candidate satisfies this bar locally, the rerank must name the
  missing physical input, measured source, compatible anchor, or formula
  evidence before any runtime owner is selected;
- use internet research only if local evidence is insufficient to prove
  the formula or measured-anchor basis. Prefer standards, manufacturer
  technical data, papers, or official software documentation over
  summaries.
