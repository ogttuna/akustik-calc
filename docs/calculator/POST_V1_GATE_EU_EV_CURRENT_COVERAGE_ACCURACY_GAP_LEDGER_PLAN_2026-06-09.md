## Purpose

Gate EU is a no-runtime numeric coverage/accuracy rerank. It exists to
avoid selecting stale closed work after Gate ET and the 2026-06-09
thick-board route-family safety follow-up.

This is not a documentation-only pass and not a source-row catalog pass.
It binds the next executable calculator step to Gate EV: a current
coverage/accuracy gap ledger that must read the implementation and
classify current gaps before any runtime values move.

The expected payoff is high for the effort because the implementation
surface has changed recently:

- Gate ER closed the direct-fixed double-leaf field/building runtime
  adapter route.
- Gate ET closed the reinforced-concrete visible-derived missing-input
  boundary.
- The thick-board safety guard closed the generic board/panel/membrane
  Auto flip into `lined_massive_wall` by surface mass alone.
- Several historical wall, floor, steel, ASTM, and visible-route gaps are
  now already live, closed repeats, or deliberately parked behind
  `needs_input` / `unsupported`.

Gate EV must therefore rebuild the current gap ledger from current gate
evidence before selecting a value-moving runtime action or owner proof.
