## Plan Iterations

Iteration 1 - current implementation probe:
Complete direct-fixed double-leaf/framed context-owned absorptive cavity
input currently has side leaf groups, cavity depth, direct-fixed support
topology, support spacing, full absorptive cavity classification, and
context-level absorber flow-resistivity ownership. Despite those physical
inputs, the engine routes it to the Gate AY advanced-wall `needs_input`
boundary instead of promoting the existing direct-fixed bridge-loss
runtime. The current unsupported probe still shows the route-local numeric
shape around `Rw 29`, `STC 29`, `C -1.4`, and `Ctr -6.2`, but those
values are not owned outputs yet.

Iteration 2 - ROI comparison:
The already-open non-direct-fixed context-owned absorber route covers
common independent/twin/resilient framed walls. The next highest ROI is
the direct-fixed sibling because it adds a new common construction
request shape with the same user-entered layer stack and absorber inputs,
and moves four owned lab outputs at once. It is higher ROI than another
coverage refresh, broad source crawling, wording-only confidence work, or
field/building polish because it turns a complete physical input into
owned calculator values.

Iteration 3 - selected bounded implementation:
Implement only the direct-fixed context-owned absorptive cavity owner.
Route family: `wall.double_leaf_framed`. Target outputs: `Rw`, `STC`,
`C`, and `Ctr`. Required physical inputs: side leaf groups, side leaf
masses, `cavity1DepthMm`, `cavity1FillCoverage full/partial`,
`cavity1AbsorptionClass porous_absorptive`, context-level
`absorberFlowResistivityPaSM2`, `supportTopology direct_fixed`,
`connectionType direct_fix` when present, and `supportSpacingMm`.
Expected scope: direct-fixed two-leaf context-owned absorptive cavity
assemblies calculate through an owned direct-fixed bridge-loss formula
path instead of falling into Gate AY `needs_input`.
