but it is not enough to claim broad calculator accuracy.

The next bottleneck is no longer "add another isolated formula corridor".
The next bottleneck is one executable accuracy loop that sees all current
measured/source evidence, separates exact overrides from holdouts, and
lets family solvers be judged by residuals instead of by fixture success.

This slice builds that loop without changing runtime values.

## Calculator-First Bar

The target is not a better label for weak answers. The target is a
calculator that can compute realistic wall and floor layer combinations.

For this plan, `low_confidence`, `screening_fallback`,
`bounded_screening`, and similar labels are not accepted supported
outcomes. They are backlog states that must be converted into one of
these outcomes:

1. exact measured/source match with same topology, metric, and basis;
2. similarity-anchored prediction with a compatible measured neighbor
   and a later-owned correction method;
3. calibrated family solver with measured holdout residuals;
4. source-absent family solver with named physics terms and an explicit
   error budget;
5. precise `needs_input` only when the required physical inputs are
   actually missing;
6. precise `unsupported` only when the route has no owned calculation
   model yet.

This distinction matters. A row that merely says "low confidence" is not
company-internal calculation coverage. It is a selected solver gap until
the project either owns the missing physical inputs and formula corridor
or proves a compatible exact/similarity/calibration path.

## External Research Grounding

The plan follows the public shape of building-acoustics calculation
rather than a catalog-only model:

- ISO 12354-1:2017 describes airborne building prediction as calculation
  models using element performance, direct/indirect flanking, and
  theoretical structural propagation. It includes detailed frequency-band
  calculation and a restricted simplified single-number model:
  https://www.iso.org/standard/70242.html
- ISO 12354-2:2017 describes impact prediction between rooms using
  measured data for direct/indirect flanking and theoretical structural
  propagation:
  https://www.iso.org/standard/70243.html
- ISO 717-1:2020 and ISO 717-2:2020 define single-number airborne and
  impact ratings from octave or one-third-octave measurement/calculation
  results, which reinforces that `Rw`, `C`, `Ctr`, `Ln,w`, and related
  outputs must stay tied to their metric basis:
  https://www.iso.org/standard/77435.html and
  https://www.iso.org/standard/69867.html
- Commercial tools make the same separation between prediction engines,
  databases, and building workflows. INSUL predicts TL, `Rw`/`STC`, and
  `Ln,w`/`IIC` from construction inputs and documents mass law,
  coincidence, double-panel, finite-size, and bridging effects; BASTIAN
  calculates EN 12354 airborne/impact building performance with ISO 717
  ratings; Acoulatis/SOAB separate element prediction from structured ISO
  12354 input data:
  https://www.insul.co.nz/features/,
  https://www.insul.co.nz/tech-info/,
  https://www.datakustik.com/fileadmin/user_upload/Bastian/Technical_Brochures/Datakustik_Bastian_Folder_EN_02-13.pdf,
  https://www.sonusoft.com/

Implication for DynEcho: source rows are exact overrides, anchors,
calibration rows, holdouts, and negative boundaries. They do not replace
the family solvers. The next executable work must expose where the solver
family is already benchmark-backed, where it is only source-absent
budgeted, and where it is still a calculation gap.
