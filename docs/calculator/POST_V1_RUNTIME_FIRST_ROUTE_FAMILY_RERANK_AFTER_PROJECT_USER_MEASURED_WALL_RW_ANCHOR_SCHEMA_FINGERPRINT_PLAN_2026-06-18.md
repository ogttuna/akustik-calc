1. Do not consume project/user measured anchors at runtime unless
   `EstimateRequestSchema` and the engine resolver expose a bounded
   active anchor input with exact fingerprint matching.
2. Do not let `draft`, `retired`, `promoted`, or conflicting anchors
   become runtime candidates.
3. Do not alias `Rw` to `STC`, `R'w`, `Dn,w`, `DnT,w`, or impact
   outputs.
4. Do not let display names, notes, visual overrides, or preset ids
   affect exact matching.
5. Do not let material physical property changes keep the same
   fingerprint.
6. Do not start UI/storage before request/resolver bridge semantics are
   selected.
