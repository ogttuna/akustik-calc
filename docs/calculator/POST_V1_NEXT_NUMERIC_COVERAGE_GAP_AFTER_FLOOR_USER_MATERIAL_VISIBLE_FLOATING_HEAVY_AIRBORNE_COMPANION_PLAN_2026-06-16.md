## Candidate Evaluation Rules

Prefer a candidate when it:

- moves real calculator outputs or closes a boundary that enables a near
  follow-up runtime move;
- uses an already-owned formula, anchor, or measured row basis;
- asks for precise physical inputs instead of guessing;
- preserves lab/field/building, ISO/ASTM, A-weighted, and companion
  metric separation;
- has a small enough contract surface to verify with current-gate.

Reject a candidate when it:

- mainly changes UI, copy, docs, process, or source gathering;
- aliases `IIC` / `AIIC` from ISO `Ln,w`;
- invents missing `CI`, `CI,50-2500`, field K, receiving-room volume,
  load basis, dynamic stiffness, or flanking inputs;
- retunes formulas without source/holdout evidence and an explicit error
  budget;
- reopens a just-closed lane without new evidence.
