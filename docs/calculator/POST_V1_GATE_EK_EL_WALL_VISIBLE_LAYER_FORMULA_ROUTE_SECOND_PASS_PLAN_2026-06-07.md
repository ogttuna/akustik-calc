## Why This Is The Highest ROI Next Move

Gate EJ closed the ASTM exact-band input surface. Explicitly labelled
ASTM E492/E989 and E1007/E989 bands now reach the owned `IIC` / `AIIC`
owner, while ISO and missing-method bands remain stopped. Repeating the
same ASTM or floor route work would not increase calculator scope.

The remaining broad calculator opportunity is wall layer-entry breadth.
The old wall coverage plan is useful context, but it predates many
post-V1 runtime/basis gates. Since then the engine has added or pinned
common flat double-leaf, explicit flat-order, grouped multicavity,
local-substitution, exact-source, timber-stud, CLT, and heavy-core wall
routes. Gate EL must therefore reconcile the old wall plan against the
current implementation before selecting any runtime movement.

This is not a source-row crawl. It is not a frontend polish pass. It is
not a confidence wording pass. The point is to find the next visible
wall layer subset where the engine can choose the right existing owned
route, or where a bounded route owner is required, while keeping
missing route-required inputs as `needs_input`.

## Candidate Filter For Gate EL

Gate EL may select a runtime/basis correction only for a candidate that
passes all of these checks:

1. The request is a realistic wall visible-layer entry, not a finite
   catalog row.
2. The route has complete physical owner inputs in the request or in
   layer-owned metadata.
3. The current implementation returns `unsupported`, `needs_input`, a
   generic boundary, or a weaker basis even though an owned route should
   handle it.
4. The selected route owns the requested metric and basis. Lab `Rw` /
   STC / `C` / `Ctr` must not become field/building `R'w` / `Dn,w` /
   `DnT,w` / `Dn,A` / `DnT,A` without an adapter owner.
5. Missing `supportTopology`, `studSpacingMm`, `resilientBarSideCount`,
   advanced-wall panel dynamics, field room context, or building context
   remains `needs_input`.
6. ASTM impact metrics remain unsupported unless the ASTM owner is
   explicitly selected. Wall airborne routes do not produce `IIC`,
   `AIIC`, `Ln,w`, `DeltaLw`, or ISO 717-2 impact outputs.
7. Exact measured rows and exact-source companion rows keep precedence.
8. Runtime movement is allowed only after before/after tests prove a
   new calculable request shape or a corrected route/basis selection.
