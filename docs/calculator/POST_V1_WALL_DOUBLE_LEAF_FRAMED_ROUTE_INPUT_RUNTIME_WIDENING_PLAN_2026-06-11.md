## Selected Route Declaration

Route family: `wall.double_leaf_framed`.

Target outputs for the first runtime widening: element-lab `Rw`, `STC`,
`C`, and `Ctr` from the owned double-leaf/framed frequency-curve route.
Field/building outputs such as `R'w`, `Dn,w`, `DnT,w`, `Dn,A`, and
`DnT,A` remain separate adapter work unless the same implementation slice
proves the required context and existing Gate I / Gate AR adapter basis.

Required physical inputs:

- side A and side B leaf layer groups;
- side A and side B leaf surface mass, derived from material density and
  layer thickness when owned, or supplied by the user when material data
  is missing;
- cavity depth and cavity layer grouping;
- cavity absorber class, fill coverage, absorber thickness, and flow
  resistivity source;
- support topology: independent frames, twin frame, shared stud,
  resilient channel, or direct-fixed;
- support spacing or stud spacing;
- resilient bar side count when support topology uses resilient channel;
- bridge class and bridge-coupling boundary;
- exact-source precedence and measured-anchor override checks.

Expected scope movement: more common board / panel / cavity / porous-fill
double-leaf/framed wall layer combinations should calculate when those
required physical inputs are present, instead of falling back to a generic
unsupported result or an over-conservative route-input stop. This is
especially high ROI for gypsum board, acoustic gypsum, cement board, OSB,
plywood, MLV/bitumen membrane, air cavity, glasswool, and rockwool
permutations where exact measured rows are expected to be absent.

Expected accuracy movement: no broad retune is allowed in this slice. The
first movement should preserve the existing source-absent error budgets
and expose a better bounded formula route by making the required physical
inputs explicit in the selected candidate trace. Any numeric retune needs
same-basis holdouts or a named bounded rule in a later slice.

Boundaries that must stay pinned:

- exact owned measured rows still win before the formula route;
- the thick-board Auto family guard remains active: board/panel/membrane
  leaves must not silently become `lined_massive_wall` or massive
  substrates from thickness or surface mass alone;
- missing support topology, missing leaf grouping, missing cavity depth,
  missing absorber properties, or missing resilient side count must return
  precise `needs_input`;
- direct-fixed double-leaf bridge-loss lanes stay on the already owned
  direct-fixed route unless the implementation explicitly selects that
  owner;
- field/building adapters remain separate and cannot alias lab values;
- ASTM/IIC/AIIC and floor impact outputs remain unsupported for this wall
  airborne route;
- grouped triple-leaf, multicavity, and calibrated local-substitution
  routes stay separate from this double-leaf/framed widening.

## Iteration 1 - Current Implementation Comparison

The engine already has an owned double-leaf/framed formula corridor and
runtime basis:
`layer_combination_resolver_double_leaf_framed_wall_banded_source_absent_formula_corridor`.
It also has direct-fixed double-leaf bridge-loss work and field/building
adapter routes. That means the next high-ROI move is not to invent a new
formula family. The implementation should widen how real layer stacks and
route-required inputs reach the existing owner while preserving its
metric-basis boundaries.

Rejected alternatives for this immediate next step:

- another compatible-anchor micro-metric owner: lower ROI after C/Ctr,
  STC, A-weighted, and building `Dn,A` lanes have already landed;
- heavy-core/lined-massive calibration: potentially valuable, but prior
  evidence/holdout plans rejected it without admissible same-basis rows;
- floor raw-bare/floating holdout tightening: also valuable, but prior
  holdout closeout rejected runtime tightening until admissible holdouts
  exist;
- broad source crawling: not calculator behavior and not needed before
  this route-input runtime widening.

## Iteration 2 - Runtime Slice Shape

The selected implementation should add a focused engine contract that
starts from currently non-calculable or over-stopped double-leaf/framed
wall requests and proves one bounded promotion path. The contract must
include:

- at least one complete board/cavity/porous-fill double-leaf/framed stack
  that was previously stopped and now calculates element-lab `Rw`, `STC`,
  `C`, and `Ctr`;
- a matching missing-input row for each required physical input that is
  newly honored;
- exact-source precedence and compatible anchor precedence checks;
- thick-board Auto family safety checks for gypsum/acoustic gypsum,
  cement board, OSB, plywood, MLV, and bitumen-like board or membrane
  leaves;
- direct-fixed, field/building, ASTM/IIC/AIIC, grouped triple-leaf, and
  floor-impact negative rows.
