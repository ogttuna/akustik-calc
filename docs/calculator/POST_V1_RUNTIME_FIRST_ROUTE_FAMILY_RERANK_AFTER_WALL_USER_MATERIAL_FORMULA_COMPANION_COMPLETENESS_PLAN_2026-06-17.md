`docs/calculator/RUNTIME_FIRST_ROUTE_FAMILY_CAMPAIGN_PLAN_2026-06-17.md`
as the selection guard. Compare route families that can move real
calculator runtime behavior or accuracy, then select one concrete owner.

  companions or field/building continuations;

  data is sufficient to expose values without scalar aliasing;
- CLT/mass-timber wall routes only if source evidence or owned physics
  is sufficient to avoid family substitution;
- heavy-core / lined-massive wall routes only if the basis can move
  beyond screening or a bounded evidence-backed delta;
- steel/floor fallback lanes only if they produce a higher runtime or
  accuracy ROI than the wall route-family candidates.

Preferred posture:

- pick a runtime owner over another no-runtime guard;
- select a source-evidence task only when it imports or validates a
  bounded measured row needed for the next formula/anchor owner;
- reject UI/report-only and process cleanup work.

## Selection Criteria

Prefer the candidate that can move runtime behavior with the strongest
calculator ROI:

- `newCalculableRequestShapes >= 2`, or
- `newCalculableTargetOutputs >= 5`, or
- `runtimeBasisPromotions >= 2`, or
- a meaningful accuracy promotion for an already common user-entered
  route.

The rerank must name:

- route family;
- target outputs;
- owned formula, spectrum owner, adapter, or measured-anchor basis;
- route-required physical inputs;
- unsupported or `needs_input` boundaries;
- estimated runtime movement.

## Boundaries

Do not select:

- UI/report-only work;
- generic process cleanup;
- source crawling without a bounded measured-row import or route evidence
  unblocker;
- confidence labeling;
- scalar metric aliasing that invents `STC`, `C`, or `Ctr`;
- field/building outputs without complete context and an owned adapter;
- ASTM/ISO metric substitution.

No-runtime output is acceptable only for this rerank because it selects
the next runtime-first owner. The following slice should move runtime
behavior unless the rerank proves a safety blocker that must be guarded
first.
